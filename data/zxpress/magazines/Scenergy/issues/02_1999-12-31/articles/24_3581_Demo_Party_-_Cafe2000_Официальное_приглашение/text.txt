                         Общие правила

 - для участия в конкурсе принимаются работы созданные для
   платформ: Amiga, PC, БК, ZX-Spectrum
 - работы сделанные для платформ Amiga и PC оцениваются вместе
 - конкурс в каждой номинации будет проводится при условии, что
   будет представлено две и более работы
 - порядок показа работ в номинациях будет определен по жребию
 - все работы должны быть сделаны специально для конкурса
 - принимаются только ранее нигде не выставлявшиеся работы
 - допускаются работы запакованные архиваторами
   rar, zip, lha, lzx
 - работы на конкурс могут представлять доверенные лица, на
   которых ложится вся ответственность за их корректное
   исполнение
 - одно доверенное лицо не может представлять более одного
   автора/группы
 - участник, занявший призовое место (1-3) в одной из номинаций
   на конкурсе, получает приз
 - лица, занявшие призовые места (1-3), но не присутствующие на
   конкурсе (участие через доверенных лиц) призы не получают
 - работы желательно прислaть зaрaнее, для тестировaния,
   и проверки нa демо-мaшинaх
 - последний срок сдачи работ - за 30 минут до конкурса
 - каждый участник должен заполнить регистрационную форму



                       Правила конкурсов

   Amiga/PC демо/интро

   Общие правила [Amiga]

 - минимальная конфигурация MC68030/68882/16 Мб. fast
 - максимальный размер рабочей памяти 32 Мб. fast
 - зацикливание только с использованием параметра "-loop"
 - работа без вмешательств со стороны пользователя
 - выход из демо/интро в систему в любой момент по нажатию левой
   кнопки мыши
 - при выходе из демо/интро компьютер не должен перегружаться,
   т.е. выход должен происходить назад в операционную систему
 - запрещена чистая анимация
 - работа после запуска SetPatch

   Общие правила [PC]

 - минимальная конфигурация P166MMX/16 Мб.
 - максимальный размер рабочей памяти 32 Мб.
 - максимальное разрешение 800x600
 - максимальная глубина цвета 24 бита (16,7 миллионов цветов)
 - зацикливание только с использованием параметра "-loop"
 - работа без вмешательств со стороны пользователя
 - выход из демо/интро в систему в любой момент по нажатию ESC
 - при выходе из демо/интро компьютер не должен перегружаться,
   т.е. выход должен происходить назад в операционную систему
 - запрещено использование аппаратных ускорителей (3Dfx и пр.)
 - запрещен сервис Windows DirectX, DirectSound и пр.
 - запрещена чистая анимация
 - допускается использование спецификации Vesa 2.0 (и UniVBE)
