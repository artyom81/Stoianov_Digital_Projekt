Stump 03 - Железо - для тех кто захочет подключить модем к Cпектруму.
─────────────────────────────────────────
      Представленная  здесь информация уже
 несколько  устарела относительно IBM сов-
 местимых машин. Мы напечатали это для тех
 кто захочет подключить модем к Спектруму.
 ─────────────────────────────────────────
                                          
      Совместные разработки  изготовителей
 модемов с группами, устанавливающими меж-
 дународные стандарты,  такими, как CCITT,
 в  последние  годы  доказали,  что модемы
 различных видов могут  взаимодействовать.
 Существует три главных области, в которых
 модемы могут быть стандартизированы:  мо-
 дуляция,  коррекция  ошибок и сжатие дан-
 ных.                                     
      Стандарт модуляции  CCITT V.32 опре-
 деляет промышленный стандарт для дуплекс-
 ной связи 9600 бит/сек. Однако V.32 явля-
 ется относительно новым  стандартом,  ут-
 вержденным в 1988 году.  Модем, соответс-
твующий лишь стандарту V.32, нельзя будет
 связать  с устройствами из огромного мно-
 жества широко распространенных модемов  с
 низкой скоростью передачи,  поэтому боль-
 шинство поставщиков пользуется также дру-
 гими,  более  старыми стандартами модуля-
 ции.                                     
      Если покупатели собираются  осущест-
 влять связь только в пределах своих собс-
 твенных частных сетей, модем должен соот-
 ветствовать  старым,  более  упрочившимся
 стандартам,  таким как Bell 212A компании
 AT&T  -  наиболее распространенному стан-
 дарту для скорости передачи 1200  бит/сек
 или V.22 бис CCITT - наиболее распростра-
 ненному  стандарту  для   скорости   2400
 бит/сек.                                 
      Хотя новые методы модуляции повышают
 скорость модемов, вероятность помех в те-
 лефонной линии,  искажающих передачу дан-
 ных, также чрезвычайно возрастает.
Рост помех может сказаться на потере
 данных  при  передаче  от одного модема к
 другому или в том,  что  в  данные  могут
 быть внесены символы, сгенерированные по-
 мехами.                                  
      Интерактивные пользователи    обычно
 видят результаты помехи в линии  передачи
 в виде необычных символов на экране.  Для
 выявления помехи в линии передачи и отка-
 за от неверно принятых данных разработаны
 протоколы передачи файлов, такие как XMO-
 DEM.                                     
      Для решения проблемы устранения  по-
 мех  в  линии  передачи компания Microcom
 Inc.,  Норвуд,  штат Массачусетс, создала
 "де-факто"  стандарт  со своим протоколом
 управления ошибкой MNP Class 4.  Протокол
 MNP Class 4 предлагается почти всеми пос-
 тавщиками модемов протокола V.32, некото-
 рые  продавцы  также  создают протоколы в
 своих модемах со скоростью передачи  2400
бит/сек.                                 
      Модемы, реализующие   протокол   MNP
 Class 4, взаимодействуют при помощи паке-
 тирования данных с использованием выявля-
 ющего ошибки кодирования.                
      CCITT также определил  протокол  уп-
 равления ошибками, V.42, который частично
 совместим (по включению) с MNP  Class  4.
 Некоторые  поставщики модемов,  соответс-
 твующих только протоколу MNP Class 4, на-
 зывают свои модемы "совместимыми с прото-
 колом V.42",  хотя на самом деле  они  не
 полностью согласуются со стандартом V.42.
      Сжатие данных может дать пользовате-
 лям  пропускную  способность канала более
 9600  бит/сек   посредством   втискивания
 большего  объема  данных  в меньшее коли-
 чество бит.  ПК передает  данные  модему,
 который,  в  свою  очередь,  осуществляет
 сжатие данных, прежде чем переслать их по
 телефонной линии.
Принимающий модем, прежде чем перес-
 лать данные в ПК, осуществляет их восста-
 новление.  В результате процедура  сжатия
 остается невидимой для пользователя.     
      Компания Microcom   также    создала
 "де-факто"  стандарт  со своим стандартом
 сжатия MNP Class 5. Для данных некоторого
 типа, особенно текстовой информации в AS-
 CII,  пропускная способность в  соответс-
 твии  с MNP Class 5 может достигать 19200
 бит/сек.                                 
      Стандарт для  сжатия V.42 бис группы
 CCITT может повысить  пропускную  способ-
 ность данных до 38400 бит/сек. Разработка
 этого стандарта была  завершена  в  конце
 1989 года.                               
      Хотя протокол  сжатия  V.42  бис  не
 полностью  совместим  с протоколом сжатия
 MNP Class 5,  большая часть  поставщмиков
 планирует   использовать  для  увеличения
 пропускной способности своих модемов  оба
метода.                                  
      Покупатели модемов  протокола   V.42
 бис должны убедиться в том, что выбранный
 ими модем будет совместим с уже  установ-
 ленными  базовыми  модемами протокола MNP
 Class 5. Однако в течение долгого времени
 протокол V.42 бис, получивший международ-
 ное признание,  будет,  очевидно, будущим
 стандартом для сжатия данных.            
      Покупатели, которые ищут устройства,
 имеющие  наибольшую  совместимость и наи-
 меньший шанс выйти из употребления, долж-
 ны  выбрать  модем,  соответствующий всем
 стандартам модуляции плюс протоколу  кор-
 рекции ошибок V.42.  Производители разра-
 ботали стандарты таким  образом,  что,  к
 примеру,  модем протокола V.32 с протоко-
 лом  коррекции  ошибок  V.42  может  быть
 подключен к модему протокола V.32,  в ко-
 тором коррекция ошибок отсутствует.      
      Итак, без  тщательного изучения воз-
можностей модема,  покупатель может обна-
 ружить,  что  он  обременен  устройством,
 имеющим характеристики,  которые не могут
 быть использованы.
