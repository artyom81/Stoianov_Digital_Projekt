Изменения в ZXRar со времён IG #5
ZXRarC29 (12.aug.06):

- в третий раз исправлено построение
  дерева из одного листа! Деревьев из 0
  листов пока не наблюдалось...
* зачёркивание файлов заменено на
  выделение.
ZXRarC28 (24.jul.06):

- если файлов на диске 125, последний
  показывался с мусором.
+ при упаковке одного файла (или при
  Separate pack) в режиме header=off
  параметр Start "архива" будет содержать
  непакованную длину, а packlist не
  создаётся.
+ в самом ZXRar и в RZXSFX05 поддержан
  выход в QC4.x (pg #d7).
- RZXSFX05: проверялось мало байт
  комстроки (Spectre)
ZXRarC27 (??.feb.05):

* при раздельной упаковке одноимённые
  архивы не стираются.
ZXRarC26 (15.feb.05):

- после прерывания Pack test не работала
  обычная упаковка (AmoNik)
+ добавлена багофича - упаковка каждого
  файла в отдельный архив (AmoNik)
+ можно включить в исходнике бумкание не
  на бипере, а на AY (AmoNik:))
ZXRarC25 (16.sep.04):

- #18 глючил при упаковке. Исправлено. Дан
  запас в 16 байт сверх минимального
  неглючащего буфера между STKEND и SP.
!совет! сжатие с 4k словарём - это не так
уж плохо, если учесть, что оно в 3 раза
быстрее...
ZXRarC24 (06.sep.04):

+ автоопределение cache. Оба варианта в
  одном флаконе.
+ #3d13 #18 для смены дисков.
+ переключаемый размер словаря для нужд
  видео (кнопка K) 1,2..32k
+ словарь 0k - pure huffman pack для
  экспериментов с энтропией. Формат
  обычный, но для распаковки не требуется
  буфер.
